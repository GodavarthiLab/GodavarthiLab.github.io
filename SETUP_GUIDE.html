Setup Guide

Publishing the
Godavarthi Lab website

Step-by-step instructions for GitHub Pages — estimated time: 30 minutes

What you'll end up with

A live website at either yourusername.github.io (free, instant) or a custom domain like godavarthilab.com (~$15/year). The site will update automatically every time you upload a new file.

No coding knowledge required beyond what's in this guide. You'll be clicking buttons on a website — nothing goes in a terminal.

Part 1 — Prepare your files

Your website folder should look exactly like this before you upload anything:

godavarthi-lab/
│
├── index.html ← homepage (rename from godavarthi_lab_website.html)
│
├── pages/
│ ├── people.html
│ ├── publications.html
│ └── news.html
│
├── images/ ← create this folder, add photos here later
│
└── css/
└── style.css
Important: The homepage file must be named exactly index.html — not godavarthi_lab_website.html. GitHub Pages only recognizes index.html as the homepage.

Part 2 — Create a GitHub account

Step 1
Go to github.com and sign up

Visit github.com and click "Sign up." Choose a username — this will appear in your URL if you don't use a custom domain, so pick something like godavarthilab or your name.

Verify your email address when prompted.

Part 3 — Create the repository

Step 2
Create a new repository

Once logged in, click the green New button (top left, or at github.com/new).

In the "Repository name" field, type exactly: godavarthilab.github.io — replacing godavarthilab with whatever username you chose.

The repository name must end in .github.io and must start with your exact username. This is what makes GitHub Pages work.

Set visibility to Public. Leave everything else as-is. Click Create repository.

Part 4 — Upload your files

Step 3
Upload index.html first

On the empty repository page, click uploading an existing file (there's a link in the middle of the page).

Drag your index.html file into the upload area, or click "choose your files" to browse for it.

Scroll down and click Commit changes (the green button). Leave the message as-is.

Step 4
Create the pages folder and upload interior pages

From your repository homepage, click Add file → Create new file.

In the filename box, type: pages/people.html — typing the slash automatically creates the folder.

Open your local people.html file in any text editor (TextEdit on Mac, Notepad on Windows), select all the text, and paste it into the large text area on GitHub.

Click Commit new file. Repeat this for pages/publications.html and pages/news.html.

Step 5
Create the images folder (for future use)

GitHub doesn't allow empty folders. To create the images/ folder now (so it's ready for your photo), go to Add file → Create new file and name it images/.gitkeep. This is a blank placeholder file that just holds the folder open.

Click Commit new file.

When you have a photo of Dr. Godavarthi, upload it to this folder and update the <img> tag in people.html — there's a comment in the file showing exactly how to do this.

Part 5 — Enable GitHub Pages

Step 6
Turn on GitHub Pages

In your repository, click Settings (top navigation bar, gear icon).

In the left sidebar, click Pages.

Under "Source," select Deploy from a branch. Under "Branch," select main and leave the folder as / (root). Click Save.

Wait about 2 minutes, then refresh the page. You'll see a message: "Your site is live at https://godavarthilab.github.io"

Step 7
Visit your live site

Go to https://yourusername.github.io in a browser. Your homepage should appear. Click through the navigation to make sure all pages load correctly.

If pages show a 404 error, double-check the filenames match exactly — GitHub is case-sensitive. People.html ≠ people.html.

Part 6 — (Optional) Add a custom domain

Step 8
Buy a domain name

Visit Namecheap.com and search for something like godavarthilab.com or godavarthilab.org. Domains cost around $10–15/year. Purchase it and create an account.

Step 9
Point the domain to GitHub

In Namecheap, go to Domain List → Manage → Advanced DNS. Delete any existing A records and add these four new ones:

Type: A Record · Host: @ · Value: 185.199.108.153
Type: A Record · Host: @ · Value: 185.199.109.153
Type: A Record · Host: @ · Value: 185.199.110.153
Type: A Record · Host: @ · Value: 185.199.111.153

Also add a CNAME record: Type: CNAME · Host: www · Value: yourusername.github.io

Step 10
Register the domain in GitHub

Back in GitHub, go to your repository Settings → Pages. Under "Custom domain," type your domain (e.g. godavarthilab.com) and click Save.

Also check the box for Enforce HTTPS once it becomes available (may take a few minutes).

DNS changes can take up to 24 hours to fully propagate, but usually work within an hour.

Part 7 — Updating the site in the future

Updating existing pages
Edit a file directly on GitHub

Navigate to the file in your repository (e.g. pages/news.html). Click the pencil icon (Edit) in the top right of the file view. Make your changes in the text editor. Click Commit changes. The site updates automatically within about 60 seconds.

Adding a photo
Upload a photo of Dr. Godavarthi

Go to the images/ folder in your repository. Click Add file → Upload files. Upload your photo — name it something simple like swetha.jpg.

Then open pages/people.html, find this comment:

<!-- Replace the placeholder below with an <img> tag... -->

Delete the line below it that says <div class="pi-img-placeholder">SKG</div> and replace it with: <img src="../images/swetha.jpg" alt="Dr. Swetha Godavarthi">

Adding a team member
Add a student card to People

Open pages/people.html. Find the .team-grid div for the appropriate section (Graduate Students or Undergraduate Researchers). Copy one of the existing .team-card blocks and update the name, role, and note text. Commit the changes.

Adding a news item
Post a lab update

Open pages/news.html. Copy one of the existing .news-entry blocks and paste it at the top of the .news-list div (newest first). Update the date, title, body text, and tag. Commit the changes.
